La réalisation de notre projet s’est faite en entière collaboration puisque nous avons réalisé chaque étape à deux.

Première semaine : 

Lors de la première séance nous avons cherché quel projet avait le plus de sens pour nous. Ayant pour objectif 
de devenir professeurs des écoles, nous avons choisi un projet en lien avec l’école. Nous avons donc opté pour l’idée d’un détecteur
de chaleur dans une salle de classe. Une fois notre idée de projet trouvée nous avons dû penser aux objectifs que nous 
souhaitions aborder. Quelles seraient les fonctionnalités de notre programme et quelle forme prendrait notre support physique. 

Semaine 2 :

Pour cette deuxième séance, nous avons dû réfléchir à la manière dont nous allions mettre en œuvre notre projet et à la façon 
de procéder pour l’écriture des codes dont nous avions besoin. Nous avons donc mené des recherches pour trouver les codes nous 
permettant d’indiquer la température mesurée ainsi que le taux d’humidité. Une fois le code entré dans Arduino, nous avons également
ajouté une ligne permettant d’écrire dans le moniteur série lorsque la température dépassait un certain seuil que nous avions choisi
au préalable. Le code affichait alors : « Il fait trop chaud ».

Semaine 3 :

Pour cette troisième séance, nous avons intégré le buzzer à notre carte Arduino Uno, puis nous avons cherché le code correspondant. 
Une fois cela fait, nous l’avons combiné avec le code écrit la semaine précédente.Nous avons ensuite rédigé les lignes de code
nécessaires pour que le buzzer se déclenche lorsque la température limite (définie comme étant trop élevée) était dépassée.

Semaine 4 : 

Ayant rencontré des problèmes avec la sauvegarde de notre code, nous avons dû reprendre le travail effectué précédemment, en 
veillant cette fois à bien le téléverser sur la carte Arduino Uno. Lors de cette séance, nous avons également créé un compte sur 
le site The Things Network. Puis, nous avons commencé à rassembler le matériel nécessaire à la bonne réalisation de notre projet.

Semaine 5 :

Pour cette cinquième semaine nous avons fait face à des difficultés avec l’écriture du sensor. Une fois le problème réglé nous avons
commencé le Lora. 

Semaine 6 :

Durant cette dernière semaine, nous avons terminé la configuration LoRa. Quelques minutes avant la présentation finale, nous avons 
réalisé qu'une fois connecté au module LoRa, notre buzzer ne fonctionnait plus. Pour notre présentation, nous avons donc dû modifier
la fonctionnalité de notre projet : lorsque la température limite était dépassée, un message nous était envoyé sur le site Tago.io.
À l’origine, l'envoi de ce message constituait une perspective d'évolution de notre projet, mais nous avons dû la transformer en 
fonctionnalité principale. Enfin, une fois notre oral terminé, le manque de temps ne nous a malheureusement pas permis de filmer 
notre support final.
